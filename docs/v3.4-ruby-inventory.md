# v3.4 Ruby Inventory and Removal Plan

Status: C.5 inventory. Every item here must be ported to Python or deleted
before the C.8 Ruby purge.

The v3.4 end state is Python-only orchestration. There is no "keep" tier:
Ruby files, Ruby shebang commands, Bundler, and Ruby CI/Docker setup are
temporary migration inputs only.

## Gems and Runtime Setup

| Source | Current dependency | Current use | Plan |
| --- | --- | --- | --- |
| `Gemfile` | `json_schemer`, `~> 2.5` | JSON Schema validation for command contracts, plugin manifests, schemas, state, and test fixtures. | Port validation to Python `jsonschema`; delete `Gemfile` and `Gemfile.lock` in C.8. |
| `.github/workflows/test.yml` | `ruby/setup-ruby@v1`, Ruby 3.3, Bundler cache | Installs Ruby for host test jobs. | Remove once C.7 ports scripts and tests. |
| `docker/Dockerfile` | source-built Ruby 3.4.7, Bundler | Runtime support for Ruby orchestration inside `eve/eve:slim` and `eve/eve:full`. | Remove language-build Ruby stage, `/opt/ruby`, `gem install bundler`, and `bundle install` in C.8. |
| `docker/Dockerfile.toolchain` | apt `ruby` | Optional toolchain image support for current Ruby scripts. | Remove Ruby package after scripts are ported. |

## C.7/C.8 Removal Checklist

1. C.6 ports SDK modules to `python/eve_sdk/` with tests for workdir,
   atomic YAML, secrets, config, state, schema, and plugin manifests.
2. C.7.a ports read-only commands: `catalog-options`, `plugin-list`,
   `instance-list`, and `instance-paths`.
3. C.7.b ports config/state commands: `config-env`, `config-save`, and
   `instance-state`.
4. C.7.c ports provider and package dispatch commands, provider command shims,
   package list/action helpers, and provider `tf-env` helpers.
5. C.7.d ports instance lifecycle commands and wrappers.
6. C.7.e ports Ruby test helpers, doctor, check-required, and Ruby fixture
   provider commands.
7. C.7.f ports or deletes remaining Ruby command surfaces, including
   `profile-resolve`, `plugins-sync`, selection helpers, integration runner,
   and embedded Ruby snippets in shell tests.
8. C.8 deletes `Gemfile`, `Gemfile.lock`, `core/sdk*.rb`, remaining Ruby
   scripts, Ruby CI setup, Ruby Docker setup, Bundler references, and any Ruby
   prerequisites in docs.
