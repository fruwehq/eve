# v3.4 Python Baseline

Ubuntu 26.04 was checked with:

```bash
docker run --rm ubuntu:26.04 sh -c 'apt-get update -qq && apt-get install -y -qq python3 && python3 --version' 2>&1 | tail -3
```

The image was available and reported:

```text
Python 3.14.4
```

## Version Policy

- Target host Python: Python 3.14, matching Ubuntu 26.04's default
  `python3` package.
- Absolute support floor: Python 3.12, matching Ubuntu 24.04 LTS host
  support.
- `requires-python`: `>=3.12,<4.0`.

The project may use Python 3.14 features only after Ubuntu 24.04 host
support is intentionally dropped in a later roadmap.

## Docker Base

- Runtime Docker base: `python:3.14-slim-trixie`.
- The official image tag was verified with:

```bash
docker manifest inspect python:3.14-slim-trixie
```

Runtime images should not build Python from source.

## Dependency Management

- Poetry remains the single dependency manager.
- Use the existing single `pyproject.toml` and single Poetry-managed
  virtual environment.
- Do not add bootstrap-specific dependency installers.

## Allowed Runtime Dependencies

Allowed runtime dependencies for v3.4 are:

- `jsonschema`
- `pyyaml`
- `textual`
- Python standard library modules

Any additional runtime dependency must be justified in the commit message
that adds it.
